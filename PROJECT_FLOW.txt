Version 1.1 - UX Enhancements

- Multi-caption output system:
  The results panel now renders 5 caption variations as independent cards with tone label, character count, individual copy action, and copy-all support.

- Hashtag grouping logic:
  Hashtags are normalized and grouped into Broad Reach, Medium Competition, and Niche/Targeted buckets using internal heuristics and fallback balancing to keep each group in an 8-12 range.

- Thumbnail preview workflow:
  Users can upload an image, crop it client-side for a square Instagram feed ratio, and preview placement in a 3x3 feed grid mockup. No image is uploaded to the server.

- Caption scoring system:
  Each formatted caption is scored from 0-100 based on hook quality, length fit, CTA presence, and hashtag count, with suggestions shown when quality is low.

Version 1.2 - Content Creation Enhancements

- Result display architecture:
  Generated output now renders in a structured section stack directly below the Generate Content button: Hooks, Caption Variations, CTA Suggestions, Hashtag Groups, Thumbnail Preview, and Carousel Generator.

- Thumbnail download logic:
  Thumbnail preview remains client-side and now supports optional text overlay plus PNG export through HTML5 Canvas with filename pattern instagram_thumbnail_TIMESTAMP.png.

- Carousel generation workflow:
  Main generation response now includes carousel slides, and a dedicated Carousel Generator tool can regenerate slide ideas from topic + niche + tone using the same Groq configuration.

- Client-side canvas export:
  All thumbnail rendering, overlay composition, and download generation happen fully in-browser with no server-side image upload.

Version 1.3 - Multi Tool Platform

- Navigation architecture:
  Added a persistent top navbar with responsive mobile menu and direct routing to Home, Caption Generator, Thumbnail Creator, and Carousel Builder.

- Multi-page structure:
  Reorganized the app into dedicated routes: /, /caption, /thumbnail, /carousel so each workflow is isolated and easier to use.

- Carousel builder workflow:
  Introduced a client-side carousel editor with built-in templates, custom layout controls, slide add/duplicate/delete/reorder operations, and per-slide editing for text, colors, image placement, and background media.

- Canvas export logic:
  Implemented reusable canvas export utilities for thumbnail PNG downloads and carousel slide exports (current slide or all slides) fully in-browser with no server image uploads.

Version 1.4 - Carousel Builder Upgrade

- Dynamic slide architecture:
  Carousel setup now starts with a slide-count modal (3-10) and initializes a matching editable slide set with unlimited per-slide image replacement.

- Template system:
  Added a dedicated template loader with reliable access to templates 1-4 plus custom layout, each with explicit image/text placeholders, background color, and grid metadata.

- Font loading:
  Implemented dynamic Google Font loading for design text layers (Poppins, Montserrat, Bebas Neue, Oswald, Playfair Display, Anton, Raleway, Lato) with live preview in editor panels.

- Canvas rendering logic:
  Added renderer utilities to draw slide backgrounds, image frames, and styled text (including gradient/highlight options) fully in browser canvas.

- Slide export workflow:
  Users can export current slide or all slides as PNG files with ordered filenames, while drag-and-drop slide strip controls export order.

Version 1.5 - Stories & Reels Toolkit

- Story template system:

- Birthday template design:

- Script and shot list logic:

Version 1.6 - Story Template Asset System

- Decorative asset system:
  Added a reusable story asset loader and curated decorative packs (balloons, confetti, cake, gift, party hat, sparkles) with local files in assets/story_templates/birthday and runtime usage via public story template paths.

- Birthday template collection:
  Expanded Story Creator with a dedicated Birthday Story Templates category including Birthday Wish, Funny Birthday, Aesthetic Birthday, Friend Birthday, and Birthday Countdown layouts, each combining background images, decorative assets, emoji accents, and shape overlays.

- Image removal functionality:
  Added image layer selection with right-panel controls for Replace Image and Remove Image; removing the image restores the placeholder so users can upload a new photo immediately.

- Asset loading logic:
  Introduced modular asset/background registries and template-decoration mapping logic to keep template metadata clean, support panel-driven asset insertion, and ensure story canvas exports include all visual layers (background, shapes, user image, decorative assets, text, emojis).

Version 1.7 - JSON Template Engine

- Template JSON architecture:
  Added a data-driven story template library at templates/storyTemplates.json with 50 templates across birthday, aesthetic, minimal, quote, poll, engagement, and promotional categories. Each template is defined through JSON elements (text, image, emoji, shape) so new templates can be added without hardcoding React components.

- Asset loading system:
  Added reusable template loading utilities in utils/templateLoader.js to read JSON templates, expose category metadata, and map JSON element definitions into the Story editor layer model (background, editable text, emojis, decorative assets, shape overlays, and primary photo placeholder).

- Canvas rendering workflow:
  Added StoryTemplateRenderer to render JSON-derived layers onto the Story canvas with full edit support (drag, resize, rotate, remove, text edit, image replace/remove) while preserving client-side HTML5 canvas export flow.

- Template category filtering:
  Updated StoryTemplatePicker with category filters and preview cards so users can browse templates by Birthday, Aesthetic, Minimal, Quote, Poll, Engagement, and Promo before applying them to the editor.
